# Zenith-Analysis

[English](./README.md) | [简体中文](./README.zh-CN.md) | [日本語](./README.ja.md)

Zenith-Analysis は、米国株、香港株、中国 A 株、日本株に対応したセルフホスト型のテクニカル分析ワークスペースです。マルチタイムフレームの市場データ、ルールベースのテクニカル根拠、買い場魅力度スコア、インタラクティブチャート、任意の LLM レポートを、1 つのレスポンシブなダッシュボードにまとめます。

本プロジェクトは調査・学習を目的としており、投資助言を提供するものではありません。また、外部の市場データについて正確性や可用性を保証しません。

## 主な機能

- 対応市場の銘柄を検索し、分析履歴と株価スナップショットをブラウザ側に保存します。
- データプロバイダーが利用できる場合、実際の市場データから日足・週足の分析スナップショットを構築します。
- 連動する価格・指標ペイン、日足/週足切り替え、赤上昇または緑上昇の配色を提供します。
- 明示的なルール根拠とデータ品質制約から、0-5 の買い場魅力度スコアを算出します。
- 設定済みの LLM が根拠をレビューし、ルールスコアを最大 `+/-0.5` 調整できます。
- 概要、戦略、テクニカル詳細を分けたレポートを生成します。
- LLM の生成に失敗し、フォールバックが有効な場合は、内蔵のローカルレポートを使用します。
- ライブデータを取得できない場合は、オフラインのデモデータであることを明示します。

## 分析範囲

| 分野 | 分析内容 |
| --- | --- |
| トレンドとボラティリティ | EMA、ボリンジャーバンド、Ichimoku Cloud、ATR、日足/週足構造 |
| モメンタム | MACD、KDJ、RSI、直近のクロスとダイバージェンス |
| 出来高と資金フロー | 出来高移動平均、OBV、CMF、価格と出来高の確認 |
| 価格位置 | 水平サポート/レジスタンス、フィボナッチ・リトレースメント、VPVR のバリューエリアと主要ノード |
| 構造とパターン | クラシカルチャートパターン、文脈を考慮したローソク足パターン、TD Sequential |
| 市場構造 | エリオット波動のヒューリスティック分析、Chanlun の分型・ストローク・中枢コンテキスト |
| 意思決定支援 | エントリーシナリオ、無効化水準、目標値、リスクリワード、データ品質によるスコア上限 |

検出対象には、ダブル/トリプルトップ・ボトム、ヘッドアンドショルダー、カップウィズハンドル、ラウンディング形状、フラッグ、レクタングル、トライアングル、ペナント、上昇/下降ウェッジが含まれます。パターンの検出結果は分析根拠の一部であり、それだけで売買シグナルになるものではありません。

## 対応市場とデータ

| 市場 | シンボル例 |
| --- | --- |
| 米国株 | `AAPL`、`MSFT` |
| 香港株 | `0700.HK`、`9988.HK` |
| 中国 A 株 | `600519.SS`、`000001.SZ` |
| 日本株 | `7203.T`、`9984.T` |

市場データの可用性は、銘柄、市場、ネットワーク、上流サービスに依存します。アプリケーションは市場と API に応じて、Yahoo Finance、EastMoney、Tonghuashun、Kabutan、Tencent、および任意の Twelve Data/FMP をフォールバック先として使用します。順序はエンドポイントや市場によって異なります。

対応市場ではリアルタイム株価を分析スナップショットへ統合し、現在値、騰落率、指標、スコアが同じ最新ローソク足を参照するようにします。最終フォールバックやシミュレーションのデータを、主要市場データとして長期キャッシュすることはありません。

## AI レポートとスコア

分析エンジンは、まず変更されないテクニカル根拠のスナップショットを構築し、決定論的な買い場魅力度スコアを計算します。トレンド、モメンタム、サポート/レジスタンス、ボラティリティ、出来高、VPVR、フィボナッチ、パターン、およびデータ品質が評価対象です。

LLM を設定すると、モデルには構造化された根拠が渡され、指標を独自に作成・再計算しないよう制約されます。ルールスコアの見直しは `+/-0.5` 以内に限定され、提示された根拠の引用が求められます。レポートは次の 3 部で構成されます。

- 市場とセットアップの概要
- 戦略、条件、リスク管理
- 詳細なテクニカル根拠

画面では Google Gemini、OpenAI、Anthropic を選択できます。OpenAI 互換サービス向けに独自の Base URL を設定することもできます。LLM が未設定の場合、またはリクエストに失敗してローカルフォールバックが有効な場合は、内蔵レポートエンジンを利用できます。

- Node.js 20.9 以降
- npm

```bash
git clone https://github.com/gunzero5158/zenith-quant.git
cd zenith-quant
npm install
npm run dev
```

[http://localhost:3000](http://localhost:3000) を開きます。


## 設定

### LLM

アプリケーションの **AI 設定** を開き、次の項目を入力します。

- プロバイダー
- モデル名
- API Key
- 任意の API Base URL
- ローカルレポートのフォールバックを有効にするかどうか

ブラウザはこれらの設定をローカルに保存します。分析時には、認証情報が本アプリケーションのサーバールートへ送られ、選択した LLM の上流エンドポイントへ転送されます。本アプリケーションが LLM の認証情報をサーバー上へ意図的に永続保存することはありません。

プライベートまたは内部ネットワーク上の LLM ホストは、初期状態では拒否されます。セルフホスト環境でプライベートエンドポイントを使用する場合は、明示的に許可してください。

```env
ZENITH_ALLOW_PRIVATE_LLM_HOSTS=true
```

### 任意の市場データプロバイダー

追加のフォールバックプロバイダーを有効にするには、`.env.local` を作成します。

```env
TWELVE_DATA_API_KEY=your_key
FMP_API_KEY=your_key
```

これらのキーは任意です。設定しなくても内蔵のプロバイダーチェーンを利用できますが、結果は上流サービスの可用性に依存します。

## プライバシーと制限事項

- 分析履歴、キャッシュ済みレポート、表示設定、LLM 設定は、ブラウザストレージまたは Cookie に保存されます。
- 市場データと AI 出力は外部サービスに依存し、遅延、不足、停止、誤りが生じる場合があります。
- デモモードの警告が表示される場合、結果はライブ市場データではなくシミュレーションデータに基づきます。
- ローカルキャッシュは重複リクエストを減らしますが、強制更新で新しい分析を要求できます。
- テクニカル指標、パターン、スコア、生成レポートは確率的な意思決定支援であり、価格予測や売買指示ではありません。
