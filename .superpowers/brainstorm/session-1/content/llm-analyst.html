大模型分析员设计方案 / LLM Analyst Integration

如何利用大模型 (LLM) 编写极具专业深度的 TradingView 风格股票分析研报？

A

硬核算法计算 + 大模型深度生成 (强烈推荐)

分工明确：数学指标（EMA, MACD 等）和几何形态（W底、神奇九转、背离）全部由程序算法客观计算。计算结果（数据结构、形态列表、打分）与基本面信息作为 Prompt 的输入，交给 Gemini 2.5 Flash / 1.5 Flash 生成分析结论。兼顾数学上的绝对准确性与自然语言的智能总结。

B

纯规则模板生成

不使用大模型，完全通过预设的条件分支（IF-ELSE）和文本模板拼装结论。虽然不需要 API Key，但分析语调生硬、死板，缺乏大模型“华尔街分析师”般的灵活思考与深度洞察。

LLM 分析员特性设计：

  • 双重模式自动切换：
    • LLM 智能模式：检测到 GEMINI_API_KEY 环境变量时，调用 Google Gemini API，生成地道的 TradingView 风格研报。
    • 规则兜底模式 (Fallback)：若未配置 Key，自动采用本地规则算法拼接分析报告，确保网站随时可用。
  • 数据输入 Prompt 示例： 
    股票代码: AAPL
当前价格: 178.58 (+1.43%)
指标状态: 日K多头排列 | MACD金叉向上 | RSI 65强势区
高级信号: 神奇九转为日5 | 简版缠论向上笔运行中
经典形态: 突破 $176.00 箱体颈线，疑似“杯柄突破”
请扮演华尔街资深量化分析师，为该股撰写一份 TradingView 风格的专业技术分析报告，重点剖析多周期共振或冲突，并给交易者具体建议。
