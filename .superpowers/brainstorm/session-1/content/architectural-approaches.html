选择分析计算层 / Analysis Calculation Layer

技术指标与形态识别算法应该运行在前端（Browser）还是后端（Next.js Server）？

A

服务器端计算模式 (强烈推荐)

计算逻辑全部放在 Next.js API 路由中。服务器抓取 K 线数据后，直接计算出 EMA/MACD/RSI、神奇九转、背离、W底等所有指标与形态，并在服务器生成最终的 0-5 评分和文本结论。前端只负责绘制图表（Lightweight Charts）和展示精美的分析面板。

B

客户端计算模式

服务器仅充当数据搬运工。Next.js API 只返回原始的 K 线数据，所有的技术指标计算、形态识别算法、打分与结论生成均在用户的浏览器中通过 JavaScript 实时计算并渲染。

A. 服务器端计算 (推荐理由)

  • 🚀 极佳的缓存性能：可以在服务器端缓存当日分析结果。如果多人查询同一股票，直接返回缓存，避免重复抓取和耗时的形态分析，极大节省雅虎 API 请求限流。
  • 🔒 算法保密与精简前端：形态识别（如缠论、波浪理论）的代码不用打包进前端，保护核心打分逻辑，且前端加载极快。
  • 💻 减轻浏览器负担：复杂的形态匹配和多周期对齐计算在服务器完成，避免低配手机或电脑浏览器卡顿。

B. 客户端计算 (缺点)

  • ⚠️ 无缓存，易被限流：每次用户刷新，都必须重新向雅虎 API 发送请求，很容易触发 API 调用频率限制。
  • ⚠️ 前端代码臃肿：需要把复杂的指标库和形态识别算法库全部打包进前端 JS 文件中，影响首屏加载速度。
